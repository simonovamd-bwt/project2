# Legal Template Drafting

Навичка для роботи над шаблонами юридичних угод у проекті Pre-legal.

## Коли застосовувати

- Створення нового шаблону угоди (NDA, договір послуг, оренда тощо)
- Ревʼю існуючого шаблону на предмет пропущених пунктів або неузгодженої термінології
- Визначення змінних полів (плейсхолдерів), які користувач заповнюватиме через UI

## Як працювати

1. **LLM-виклики через стандартний Anthropic API** — ключ береться з `ANTHROPIC_API_KEY` в `.env`. Без OpenRouter, без Cerebras, без стрімінгу для коротких структурованих відповідей (генерація пункту угоди — це не чат, стрімінг тут не потрібен).
2. **Кожен шаблон угоди** повинен мати:
   - Чіткий перелік розділів (преамбула, сторони, предмет угоди, права та обовʼязки, відповідальність, строк дії, підписи)
   - Явно позначені плейсхолдери у форматі `{{ field_name }}` для полів, які заповнює користувач
   - Мову за замовчуванням — українську (проект орієнтований на укр. ринок, якщо не вказано інше)
3. **Дисклеймер обовʼязковий**: кожен згенерований документ — це draft для подальшого юридичного рев'ю, не є остаточною юридичною консультацією. Це має бути видно і в самому шаблоні, і в UI.
4. **Не вигадувати юридичні норми** — якщо шаблон посилається на конкретну статтю закону чи нормативний акт, це має бути позначено як TODO для перевірки юристом, а не видано як факт.

## Структура шаблону (референс)

```
{{agreement_title}}

Дата: {{date}}
Сторона 1: {{party_1_name}}, {{party_1_details}}
Сторона 2: {{party_2_name}}, {{party_2_details}}

1. Предмет угоди
{{subject_matter}}

2. Права та обов'язки сторін
...

3. Відповідальність
...

4. Строк дії та розірвання
...

5. Підписи
```

## Межі навички

Ця навичка **не** замінює юридичну експертизу і **не** приймає рішень про юридичну коректність документа — вона структурує чернетку. Фінальне рішення про використання документа — за користувачем/юристом.
