# Tasks

Локальний беклог фіч. Кожна задача виконується окремо, по черзі, щоб уникати перевантаження контексту агента.

## Task 1: Базова інфраструктура (еквівалент PL4)

- **Архітектура:** створити базовий проєкт — FastAPI (менеджер `uv`) для бекенду та статичний Next.js для фронтенду.
- **Деплой:** упакувати фронтенд і бекенд у єдиний Docker-контейнер через multistage build.
- **База даних:** SQLite, яка створюється з нуля при кожному запуску контейнера. Потрібна на майбутнє для збереження користувачів; автентифікація зараз — лише заглушка (placeholder only).
- **Функціонал:** форма для угоди про нерозголошення (NDA); робота з даними форми — виключно на клієнті (client-side), без збереження на бекенді.

## Task 2: Інтеграція ШІ чат-інтерфейсу (еквівалент PL5)

- **UI/UX:** повністю замінити статичну форму (з Task 1) на інтерактивний ШІ чат-інтерфейс.
- **Логіка діалогу:** ШІ сам розпочинає бесіду (вітається) і ставить перше запитання. Коли всі необхідні поля зібрано — підтверджує це й показує кнопку завантаження PDF.
- **Прев'ю:** документ оновлюється наживо (live preview) паралельно з відповідями користувача в чаті.
- **Технічний дизайн ШІ:** бекенд робить один виклик до LLM без стрімінгу (no streaming); відповідь моделі — у форматі structured outputs, щоб одночасно отримати текст для чату та екстраговані поля для документа.
